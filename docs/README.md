# Documentation

This folder contains **living documentation** that is always kept up-to-date with the current state of the project.

## What Belongs Here

| Type | Examples |
|------|----------|
| Getting Started | How to install, configure, run |
| API Reference | Current function signatures, parameters |
| Configuration | Valid config options, defaults |
| Architecture | Current system design (update when code changes) |

## What Does NOT Belong Here

Static exploration documents belong in `notes/`:
- Design explorations
- Blog posts and essays
- Planning documents
- Comparisons written at a specific point in time

See `CLAUDE.md` for the full docs/ vs notes/ convention.
